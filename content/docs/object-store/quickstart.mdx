# Object Store Quickstart

<Steps>

<Step>

### Install The Package

```bash
composer require vortos/vortos-object-store
```

Install `vortos/vortos-docker` as well when you want `vortos:worker:*` supervisor management commands.

</Step>

<Step>

### Register The Package

```php title="bootstrap/app.php"
use Vortos\ObjectStore\DependencyInjection\ObjectStorePackage;

$packages = [
    // ... other packages
    new ObjectStorePackage(),
];
```

</Step>

<Step>

### Set Environment Variables

Cloudflare R2:

```dotenv title=".env"
VORTOS_OBJECT_STORE_DRIVER=s3
OBJECT_STORE_PROVIDER=r2
OBJECT_STORE_REGION=auto
OBJECT_STORE_ACCOUNT_ID=your-cloudflare-account-id
OBJECT_STORE_BUCKET=your-bucket
OBJECT_STORE_ACCESS_KEY_ID=your-r2-access-key
OBJECT_STORE_SECRET_ACCESS_KEY=your-r2-secret-key
OBJECT_STORE_TEMPORARY_KEY_PREFIX=tmp
OBJECT_STORE_PUBLIC_BASE_URL=https://cdn.example.com
```

AWS S3:

```dotenv title=".env"
VORTOS_OBJECT_STORE_DRIVER=s3
OBJECT_STORE_PROVIDER=s3
OBJECT_STORE_REGION=us-east-1
OBJECT_STORE_BUCKET=your-bucket
OBJECT_STORE_ACCESS_KEY_ID=your-access-key
OBJECT_STORE_SECRET_ACCESS_KEY=your-secret-key
OBJECT_STORE_TEMPORARY_KEY_PREFIX=tmp
```

Development:

```dotenv title=".env"
VORTOS_OBJECT_STORE_DRIVER=log
OBJECT_STORE_BUCKET=dev-bucket
```

Testing:

```dotenv title=".env.test"
VORTOS_OBJECT_STORE_DRIVER=null
OBJECT_STORE_BUCKET=test-bucket
```

</Step>

<Step>

### Create Config

```php title="config/object_store.php"
<?php

declare(strict_types=1);

use Vortos\ObjectStore\DependencyInjection\VortosObjectStoreConfig;

return static function (VortosObjectStoreConfig $config): void {
    $config
        ->driver($_ENV['VORTOS_OBJECT_STORE_DRIVER'] ?? 'log')
        ->provider($_ENV['OBJECT_STORE_PROVIDER'] ?? 'r2')
        ->region($_ENV['OBJECT_STORE_REGION'] ?? 'auto')
        ->endpoint($_ENV['OBJECT_STORE_ENDPOINT'] ?? null)
        ->bucket($_ENV['OBJECT_STORE_BUCKET'] ?? '');

    $config->client()
        ->accountId($_ENV['OBJECT_STORE_ACCOUNT_ID'] ?? null)
        ->credentials(
            $_ENV['OBJECT_STORE_ACCESS_KEY_ID'] ?? null,
            $_ENV['OBJECT_STORE_SECRET_ACCESS_KEY'] ?? null,
        );

    $config->bucketConfig()
        ->temporaryKeyPrefix($_ENV['OBJECT_STORE_TEMPORARY_KEY_PREFIX'] ?? 'tmp')
        ->publicBaseUrl($_ENV['OBJECT_STORE_PUBLIC_BASE_URL'] ?? null);
};
```

</Step>

<Step>

### Run Migrations

The object-store package ships the `object_store_outbox` migration. Run your normal project migration command after package registration.

</Step>

<Step>

### Install The Outbox Worker

```bash
php bin/console vortos:worker:install --worker=object-store-outbox-relay
```

</Step>

<Step>

### Plan Temporary Upload Cleanup

```bash
php bin/console vortos:object-store:lifecycle plan
php bin/console vortos:object-store:lifecycle apply --confirm
```

</Step>

</Steps>

<Callout type="warn" title="Lifecycle commands mutate provider infrastructure">
Vortos never applies lifecycle rules during HTTP requests, worker boot, or container compilation. Apply lifecycle changes explicitly in deployment or operations workflows.
</Callout>

## Generate A Direct Upload Intent

```php
use Vortos\ObjectStore\Contract\DirectUploadManagerInterface;
use Vortos\ObjectStore\ValueObject\TemporaryUploadUrlOptions;

final class CreateUploadIntentHandler
{
    public function __construct(private readonly DirectUploadManagerInterface $uploads) {}

    public function handle(CreateUploadIntent $command): UploadIntentDto
    {
        $intent = $this->uploads->createUploadIntent(
            temporaryKey: sprintf('tmp/%s/%s', $command->userId, $command->fileName),
            options: TemporaryUploadUrlOptions::forDirectUpload(
                ttlSeconds: 900,
                contentType: $command->contentType,
                maxSizeBytes: $command->maxSizeBytes,
            ),
        );

        return UploadIntentDto::fromIntent($intent);
    }
}
```
